# Projeto EDB I – Gerenciamento de Pedidos de Restaurante

## Sobre o Projeto

Projeto referente à disciplina **Estruturas de Dados Básicas I (EDB I)**, do curso de **Bacharelado em Tecnologia da Informação (BTI)** da **Universidade Federal do Rio Grande do Norte (UFRN)**, realizada em **dupla**.

O projeto consiste na implementação, em **linguagem C**, de um **sistema de gerenciamento de pedidos de um restaurante**, com foco na aplicação prática de estruturas de dados lineares.

---

## O que o Programa Faz

O sistema simula o fluxo de pedidos de um restaurante, separando as responsabilidades entre o **salão** e a **cozinha**, utilizando diferentes estruturas de dados:

- Os pedidos realizados no **salão** são armazenados em uma **lista ligada**;
- É possível **remover pratos específicos** da lista de pedidos antes do envio à cozinha;
- Os pedidos são encaminhados para a **cozinha**, onde passam a ser processados em **ordem de chegada**, utilizando uma **fila**;
- A cozinha processa os pedidos respeitando o princípio **FIFO (First In, First Out)**.

Essa abordagem permite visualizar, de forma clara, o uso e a diferença entre **listas ligadas** e **filas** em um contexto prático.

---

## Objetivos

- Implementar um sistema de gerenciamento de pedidos de restaurante em C  
- Armazenar pedidos utilizando uma **lista ligada**  
- Permitir a remoção de pedidos específicos do salão  
- Processar pedidos em ordem de chegada por meio de uma **fila**  
- Aplicar conceitos fundamentais de **estruturas de dados básicas**

---

## Estruturas de Dados Utilizadas

- Lista Ligada  
- Fila  

Essas estruturas são utilizadas para organizar e manipular os pedidos de forma eficiente, simulando um cenário real de atendimento e processamento.
